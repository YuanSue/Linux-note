WAS無法自動帶起

手動啟DB2

正常開啟VM會帶DS起來，若無法自動帶起，且用手動開啟，還是無法正常啟動，則有很大的機率是DB2沒有正常啟動
手動開啟WAS步驟:

  1. 先手動讓DSEngine啟動 ⮞
  2. 切到db2inst1: su - db2inst1讓db2啟動: db2start 如果成功會看到：SQL1063N DB2START processing was successful. ⮞
  3. 確認 Database Manager 是否跑起來: db2 get dbm cfg | head -5 ⮞
  4. 確認進程: ps -ef | grep db2sysc ⮞
  5. 列出 active databases: db2 list active databases(如果成功會看到： SQL1611W No data was returned by Database System Monitor.) ⮞
  6. restart: cd /u01/IBM/InformationServer/ASBServer/bin/ ⮞ ./MetadataServer.sh restart

WAS自動啟檢查及步驟

若要讓 WAS 自動帶起，先確認 SELINUX 是否為 disable，更改步驟如下:

  1. 確認當前 SELinux 狀態: getenforce# 通常會有三種回傳: Enforcing (強制)、 Permissive (寬鬆)、 Disabled (關閉)，確認是否為Disabled
  2. 若無，先讓狀態為Permissive -> setenforce 0，再確認是否變成 Permissive -> getenforce
  3. 編輯 vi /etc/selinux/config，將enforcing改成disable -> SELINUX=disabled
  4. 儲存後重開機 -> reboot
  5. 開機後再確認狀態是否為disable -> getenforce
  6. 開啟時間會較長，可先查看WAS服務是否有啟動，或者查看log:
    • tail -f /u01/IBM/WebSphere/AppServer/profiles/InfoSphere/logs/server1/startServer.log
      • 或者
    • tail -f /u01/IBM/WebSphere/AppServer/profiles/InfoSphere/logs/server1/stopServer.log
      • 想查看最新 100 行可用tail -n 100 ...
      又或者看SystemOut / SystemErr，WebSphere 的 server JVM 在啟動、部署、例外時都會寫到這些檔案
    • tail -n 100 /u01/IBM/WebSphere/AppServer/profiles/InfoSphere/logs/server1/SystemOut.log
      • SystemOut.log：正常啟動訊息、系統訊息（看到 “open for e-business” 代表成功啟動）。
    • tail -n 100 /u01/IBM/WebSphere/AppServer/profiles/InfoSphere/logs/server1/SystemErr.log
      • SystemErr.log：錯誤訊息、例外堆疊。
  7. 若還是無效加上 kernel 參數防呆 -> grubby --update-kernel=ALL --args="selinux=0" -> 再reboot後檢查狀態
  8. 若真的還是再無效可能標籤還是錯誤，導致啟動腳本異常，需要恢復預設標籤:
      這一步是讓系統的安全上下文回到預設值，避免錯誤標籤阻擋執行。
    • restorecon -Rv /u01/IBM/InformationServer
    • restorecon -Rv /u01/IBM/WebSphere/AppServer
      • 檢查檔案/執行檔標籤:
    • ls -Z /u01/IBM/InformationServer/ASBServer/bin/MetadataServer.sh
      • 或整個目錄:
    • ls -lZ /u01/IBM/InformationServer/ASBServer/bin
另外一種方法，修正標籤為 bin_t，目的是使用標籤繞過SELinux檢查，詳細方法
重開之後檢查log確認WAS是否正式被自動帶起
log查詢路徑: cd /u01/IBM/WebSphere/AppServer/profiles/InfoSphere/logs/server1
當中有 server1.pid 表示啟動時有建立 PID 檔
